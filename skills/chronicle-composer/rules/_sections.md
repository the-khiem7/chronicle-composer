# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Commit Analysis (analysis)

**Impact:** HIGH
**Description:** Techniques for analyzing git diffs and breaking them into logical, atomic changes.

## 2. Staging Strategy (staging)

**Impact:** HIGH
**Description:** Best practices for staging changes using hunks instead of entire files.

## 3. Commit Convention (convention)

**Impact:** HIGH
**Description:** Rules for writing meaningful commit messages that follow standard conventions.

## 4. Timeline Control (timeline)

**Impact:** MEDIUM
**Description:** Managing commit dates and maintaining chronological development order.

## 5. History Management (history)

**Impact:** MEDIUM
**Description:** Techniques for maintaining clean, readable git history that tells the project story.
